Data Science Project — Motorsport Telemetry

Every corner leaves a signature.

A telemetry analysis pipeline built on real Formula 1 sensor data — speed, throttle, brake and gear, sampled several times a second — to find exactly where one driver beats another.

speed throttle brake
01 Five analyses, in order of depth
1
Speed Trace Comparison
Two drivers' fastest laps, overlaid speed-vs-distance — see exactly where one pulls ahead.
2
Braking Point Detection
The exact track distance where each driver hits the brakes into every corner.
3
Gear & Throttle Map
Gear selection and throttle percentage across the lap — racing line and corner-exit aggression.
4
Micro-Sector Dominance
The lap split into ~25 mini-sectors, colored by who was faster in each — the same chart F1 broadcasts show live.
5
Tire Degradation & Strategy
Lap time vs. lap number across a race stint, revealing tire wear and strategy windows.
02 How to see it move

Static charts from telemetry_analysis.py, an interactive browser dashboard from dashboard.py, or two dots physically racing around the real track shape from animation.py.

# interactive dashboard streamlit run dashboard.py # animated GIF of the lap python animation.py
